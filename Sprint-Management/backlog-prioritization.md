# Backlog Prioritisation

## Objective

Prioritise SaaS MVP features and operational tasks based on business value, user impact, technical dependencies, and release readiness.

---

# Prioritisation Framework

Features are prioritised using:
- business impact
- user value
- technical complexity
- operational risk
- MVP alignment

---

# Priority Levels

| Priority | Description |
|---|---|
| P0 | Critical for MVP launch |
| P1 | High-value post-MVP feature |
| P2 | Operational improvement |
| P3 | Future enhancement |

# Prioritisation Risks

## Scope Creep
Additional features may impact the MVP delivery timeline.

### Mitigation
Strict sprint scope review process enforced.

---

## Technical Dependencies
Third-party integrations may delay implementation.

### Mitigation
Early API validation and dependency tracking.

---

# TPM Responsibilities
- manage backlog prioritisation
- align stakeholders on MVP scope
- track delivery dependencies
- prevent sprint overload
- coordinate roadmap planning
