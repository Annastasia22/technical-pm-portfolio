# Jira Workflow — SaaS Delivery Pipeline

## Objective

Define the standardised Jira workflow used to manage Agile delivery across product, engineering, QA, and release coordination teams.

---

# Workflow Stages

## Backlog
We have new feature requests, bugs, and operational tasks waiting for prioritisation.

### Examples
- feature requests
- technical debt
- QA improvements
- API enhancements

---

## Ready for Development
Requirements clarified and approved for engineering implementation.

### Requirements
- PRD completed
- acceptance criteria defined
- dependencies identified

---

## In Progress
The engineering team is actively developing the feature or fix.

### Activities
- implementation
- API integration
- frontend development
- backend development

---

## Code Review
Peer review performed before deployment to staging environment.

### Validation Areas
- code quality
- security checks
- implementation accuracy

---

## Ready for QA
Feature deployed to staging and ready for validation.

### Requirements
- deployment completed
- release notes updated
- test environment stable

---

## QA Testing
QA team validates:
- functionality
- regression impact
- edge cases
- API behaviour

---

## Ready for Release
QA approved and validated for production deployment.

### Requirements
- critical bugs resolved
- regression completed
- stakeholder approval received

---

## Done
Feature successfully released to the production environment.

### Final Activities
- release monitoring
- analytics validation
- post-release verification

---

# Bug Workflow

## Open
Bug identified and documented.

## Investigating
Engineering is reviewing the root cause.

## Fix In Progress
The bug is actively being resolved.

## Ready for Retest
QA validates the fix in staging.

## Closed
Bug resolved successfully.

---

# TPM Responsibilities
- coordinate workflow progression
- remove delivery blockers
- manage sprint priorities
- track release risks
- communicate project status
